CDK with SSO

March 18, 2021 One-minute read
development
AWS • CDK • SSO

CDK usually authorizes with default profile of the AWS CLI (on Linux systems you’ll find it in the file ~/.aws/config). If you’d like to use another profile you’re able to pass it with the --profile <yourProfile> parameter. However, if you pass an SSO profile you’ll get an error like this:

Need to perform AWS calls for account <AWSAccountID>, but no credentials have been configured

On the corresponding github issue there are several solutions mentioned. I defenitely prefered the one with aws2-wrap.
