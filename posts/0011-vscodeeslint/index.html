ESLint in VSCode

March 23, 2021 2-minute read
development
js • JavaScript • TypeScript • VSCode • npm

If your working with with JavaScript/TypeScript and you’ve not heard about ESLint so far, you might want to continue with reading.

What is it?

If you integrate it in your IDE, it helps you to

“Find and fix problems in your JavaScript code”

https://eslint.org/

For me, it’s about standardizing my source code and thus helping others to read it. Additionally, it can save me from making mistakes. It marks the code that does not comply with the defined rules and suggests appropriate corrections.

How to install and integrate it in your project?

If you are using VSCode, you can first install the extension ESLint.

Now, install the ESLint package with npm into your project. If you want to install it globally, append the parameter -g. I prefer not to do so.

npm install eslint --save-dev

With the following command you can create a configuration file. The configuration file is called .eslintrc.js and is placed in the root directory of your project.

npx eslint --init

This command will ask you some questions about your project. E.g. if you use a framework, or if you work with TypeScript. Find more information in the ESLint documentation.

Typescript

For working properly with TypeScript, I had to modify my VSCode settings.json.

Open the settings with CTRL + SHIFT + P and type > Preferences: Open Settings (JSON). Then append the following to your settings:

    "eslint.validate": [
        "typescript", "typescriptreact"
    ]

Rules

You are allowed to modify the rules by which ESLint checks your source code according to your needs. For example, I prefer to append the following rules to the project configuration file:

  rules: {
    semi: [1, 'always'],
    'key-spacing': [1, { mode: 'minimum' }]
  }
  • semi
    This rule asks for a semicolon (;) at the end of every statement. This is something that I once decided is the best approach for me.
  • key-spacing
    I prefer to be allowed to put in as many spaces as I want while defining a object literal. E.g.
const obj = {
    "one":      1,
    "seven":    7
}

And by the way, the number 1 in the rules tells ESLint that a violation of the rule is only a warning. If you want it as error put in the number 2.
